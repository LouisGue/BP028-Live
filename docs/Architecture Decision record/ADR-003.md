# ADR-003 : Paquets ajoutés au chroot post-bootstrap
## Contexte
Après le bootstrap minbase (ADR-002), le chroot ne contient que les paquets de priorité `required` et les dépendances d'`apt`. Avant de commencer le durcissement, certains paquets doivent être ajoutés. Chaque ajout élargit la surface du système et doit être justifié individuellement, conformément à BP-028 R56.

## Paquets installés
| Paquet | Catégorie | Justification |
|---|---|---|
| `ca-certificates` | Sécurité | Nécessaire pour passer les sources APT en HTTPS. Sans lui, `apt` ne peut pas valider les certificats TLS des miroirs. |
| `vim` | Développement | Éditeur utilisé pendant la phase de construction pour éditer les fichiers de configuration dans le chroot. Non requis par le système durci lui-même. |

## Paquets explicitement exclus à ce stade
| Paquet | Raison de l'exclusion |
|---|---|
| `curl` | Aucun besoin identifié. Les téléchargements passent par `apt`. |
| `gnupg` | Déjà présent via les dépendances d'`apt` dans minbase. Pas d'ajout supplémentaire nécessaire. |
| `locales` | Aucun besoin fonctionnel pour un système durci headless. Sera réévalué si nécessaire. |

## Commande retenue
```bash
chroot ./chroot apt update
chroot ./chroot apt install -y ca-certificates vim
```

## Conséquences
- La surface installée reste minimale : deux paquets ajoutés, chacun justifié
- `vim` est assumé comme outil de développement, pas comme composant du système durci ; il pourra être retiré dans l'image finale si nécessaire
- Le passage des sources APT en HTTPS devient possible après installation de `ca-certificates`
- Tout paquet futur fera l'objet d'une entrée dans ce document ou d'un ADR dédié
